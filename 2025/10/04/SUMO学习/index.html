SUMO学习

SUMO学习笔记

因研究需要，一直在找除了VISSIM之外的更合适的交通仿真软件（当然不是VISSIM不好用，首先它是商业版，更重要的是网上下载的现有路网文件经常会报某些模块的license没有授权，运行不起来）。考虑成本和时间问题，决定开始学习使用SUMO。也曾花钱报了SUMO的线上培训班，但由于时间关系，学的不是特别的专注。最近开始重新拾起，并逐步深入理解，将学习的内容也记录下，为自己以后使用、温习提供素材。

SUMO基本介绍

SUMO是德国宇航中心（DLR）研发的一款开源交通仿真软件，使用SUMO可以实现以下工作：

  • Build a Network and assign routes within a network
  • Develop traffic signal control algorithm
  • Simulate and analyze traffic within a network, given Origin-Destination (OD) Matrix
  • Etc.

SUMO的目录结构

  • bin: 可执行文件（windows)
  • data: 数据定义和格式文件
  • doc: 帮助文档
  • tools: 附加工具

SUMO仿真的输入/输出：
一般交通仿真需要的输入数据：路网结构、基础设施（信号控制器、信号灯、检测器、…)、交通需求（交通量）等。对应到SUMO里，这些都是以xml文件的方式保存，如下：

  • 路网文件 *.net.xml
  • 交通需求文件 *.rou.xml 车流行驶信息的存储文件，可以使用duarouter.exe文件生成
  • 车辆起讫点文件 trip.xml trip是按照单个车辆进行存储，因此支持更加精细的仿真
  • 车辆起讫点文件 flow.xml flow文件是将相同起讫点的车辆按照交通流存储
  • 附加元素文件 *.add.xml
  • sumo仿真配置文件 *.sumocfg

而SUMO仿真输出数据也可以分为三大类：

  • 宏观资料
  • 中观资料
  • 微观资料：每辆车运行的资料

SUMO中支持的交通模型包括：跟驰模型、换道模型、交叉口模型：当车辆靠近交叉口时，如车辆靠近交叉口的加减速行为、通过交叉口的让行行为等等。

认识SUMO-GUI

路网创建

在SUMO中，路网可以由以下三种方式创建：

  • 依据node, edge, route, connection files手工创建
  • 使用netgenerate命令
  • 从外部诸如OSM、VISSIM、VISUM中导入
  • 使用netedit绘制路网

手工创建路网

我们知道，路网一般是由节点（路口）和边（路段）组成，在SUMO中，节点和边都是由单独的文件来定义，如下：

  • 节点文件 (.nod.xml)
  • 边文件 (.edg.xml)
  • 边类型文件 (.type.xml)
  • route file 交通需求文件 (.rou.xml)
    定义好以上文件后，可以通过netconvert命令转换路网
    netconvert --node-file <*.nod.xml> --edge-files <*.edg.xml> -t <*.type.xml> -o <*.net.xml>

然后可以定义.sumocfg文件，定义好后运行如下指令启动打开建立好的路网：
sumo -c <*.sumocfg>
或者
sumo-gui -c <*.sumocfg>

从OSM中导入路网

从OSM地图中导入路网的一般步骤如下：

  1. 确定区域并从OSM中下载地图数据
  2. 将地图数据转换为SUMO路网
    netconvert --osm-files <*.osm> -o <*.net.xml>
  3. 利用内置的randomTrips.py添加trip和route
    python <SUMO_PATH>\randomTrips.py -n <*.net.xml> -r <*.rou.xml> -e 50 -l

定时控制

使用额外的配时文件 *.add.xml
加载配时文件的命令为：
sumo -a <*.add.xml> ... <other options for network and routes>
或
sumogui -a <*.add.xml> ...
这样就会覆盖之前在路网里设置好的信号配时

推荐使用NetEdit设置配时，这样state中的每个位代表哪个connection比较容易对应

界面中查看信号配时，右键信号灯，选择Show Phases

其中，detectorRange指定从停止线测量的上行检测范围（以米为单位）。默认情况下（如果参数未定义），假设基础 E2 detector 完全覆盖第一个接近通道。一旦车辆进入探测器范围，就会累积车辆的时间损失。如果其累计时间损失超过 minTimeLoss（当前默认值为1秒）的值，则如果它处于活动状态，则请求相应绿灯相位延长。
车辆的瞬时时间损失定义为 1-v / v_max，其中v是其当前速度，v_max是允许的最大速度。

执行仿真

命令行执行（需要路网文件和交通需求文件）：

sumo -C saved.sumocfg
